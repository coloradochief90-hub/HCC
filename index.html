App Instructions
How to use the Hendry Command Center
Daily Flow

1. Open HCC — lands on Daily Briefing. Review family schedule and quick-nav to any family member.

2. Go to David → Command Center — review personal streaks, trading snapshot, and failure tracker. Hard gate at the bottom must be passed before accessing any David sub-domain.

3. Pass the gate — "I'm Ready — Enter" clears once per day via localStorage. Resets automatically at midnight.

4. Trade — access Trading Command Center for pre-session checklist, market condition, loss limit, and quick links to Handbook and Data Log.

5. Log trades — David → Daily Log → Trading section. One trade at a time. Submit clears form for next trade.

6. Log diet and fitness — Daily Log → Diet and Fitness sections. Diet meters update live as you type. Fitness toggles track goal completion.

7. Evening — check HCC Completion for any pending build tasks. Review domain focus cards.

Navigation

Sidebar — tap the ☰ menu button on mobile. On desktop the sidebar is always visible. Domain names with › expand to show sub-sections.

Accent colors — every domain has its own neon accent color. The topbar title and page headers shift to that color when you navigate into a domain.

Claude + Gemini buttons — bottom of sidebar. Tap to open either tool directly.

Back to briefing — tap Daily Briefing in the sidebar at any time.

Daily Log — How It Works

Trading — fill in all fields for one trade. Tap Submit — form clears and shows confirmation. Repeat for each trade. Data will push to Trading Performance Log Google Sheet when integration is live.

Diet — enter macros and the meters appear automatically showing actual vs target. Tap Log Diet when done.

Fitness — tap toggle buttons for Exercise, Iso Holds, Stretch — they turn neon blue when on. Select workout type, enter steps, flights, intensity minutes, and body battery. Tap Log Fitness.

Updating the App

HCC is hosted on GitHub Pages. When Claude builds an updated version:

1. Download the new HTML file from Claude

2. Save as index.html in your local GitHub Desktop folder (replaces old file)

3. Open GitHub Desktop → Commit → Push Origin

4. Wait 60-90 seconds — both iPhone and desktop PWA update automatically

Data & Logging
What gets logged where and how
Google Sheet Structure

Trading Performance Log — Daily Trade Log tab (raw trade data, one row per ticker per session). App calculates Weekly, Monthly, YTD automatically. No manual compilation tabs needed.

Market Performance Log — Big Movers tab + Sector Performance tab. Updated 1-3x per week. Contains trend dividers marking Bear/Bull market regimes.

HCC Daily Log — one file, separate tabs per person. Trading tab, David tab (diet/fitness), Taylor tab, Adley, Evie, Callan, Cooper. HCC is the input interface, Sheets is storage.

Morning Prep + Strategy Reference — reference documents only, not spreadsheet tabs. Morning Prep lives as HCC checklist. Strategy Reference lives in Handbook.

Trade Log Column Structure

Section 1 — Trade Context (David inputs): Date, Day, Market Type, Market Condition, Ticker, Float, ETB/HTB, Catalyst

Section 2 — Technicals & Trade (David inputs): VWAP, 100K/Min, Trade Type, Entry Type, Initial R:R, Executed R:R, # Trades, # Green, # Red, P&L, Wash Sale Y/N, Wash Sale $ Impact

Section 3 — Analysis (David inputs): Entry timing, stop placement, what I did well, one improvement, best case P&L, sold appropriately, why I sold, my conclusion

Claude columns: Coach's Analysis, Wash Sale $ Impact calculation. Provided as copy-paste block during daily review session.

Diet & Fitness Log Fields

Diet targets: Calories 2,600-2,900 kcal · Protein 180-200g · Carbs 150-200g · Fat 70-90g · Sodium under 2,000mg · Added Sugar under 30g

Fitness fields: Exercise Y/N · Iso Holds Y/N · Stretch Y/N · Workout Day (Push/Pull/Bike/Hike/Run/Beaver Ranch) · Daily Steps · Flights of Stairs · Intensity Minutes · Body Battery

Review Cycle

Weekly — Sunday evening. Claude reads all active domain logs from Sheets. Generates cross-domain review covering trading performance, diet/fitness progression, family focus areas. Includes projections based on current trajectory. Gemini Weekly Relay follows — key data fed back into Gemini to calibrate next week's prep.

Monthly — last day of month. Full performance review all domains. Body and mind progression narrative for diet/fitness.

Quarterly — YTD check-in. Trajectory analysis and projections.

Year-end — full annual review across all domains.

Error Recovery
Mistakes, undo, and data protection
Trade Log Errors

Before submit: Review all fields before tapping Submit. Once submitted, data writes to Google Sheets.

After submit — minor error: Open the Trading Performance Log in Google Sheets directly and edit the cell. Google Sheets saves instantly.

After submit — major error: Google Sheets keeps full version history. File → Version History → See Version History. Roll back to any previous version.

Soft delete (coming Phase 2C): Rows will be marked as voided rather than deleted — data stays but doesn't count in calculations.

Calendar & Task Errors

Deleted calendar event: Goes to Google Calendar trash. Open Google Calendar → Other Calendars → Trash → Restore. Available for 30 days.

Deleted task: Google Tasks doesn't have a trash — deletion is permanent. HCC will show a confirm dialog before deleting any task.

Wrong event edit: Google Calendar keeps edit history — right-click event → See edit history to restore previous version.

App Data (localStorage)

Gate state: Stored in browser localStorage with today's date as key. Resets automatically at midnight. If gate doesn't reset — clear browser cache or use a different browser session.

Checklist state: HCC Completion checkbox states stored in localStorage. Persist between sessions. To reset — clear site data in browser settings.

Form data: Daily Log forms do not save drafts — if you navigate away before submitting, data is lost. Complete and submit each section before navigating.

Project Life Connect
Commercial life OS — bridges Apple, Google, Microsoft, and everything else
The One-Line Pitch
"One app that reads and writes across everything you already use — Google, Apple, Microsoft, Garmin, and more — organized around your actual life."
The Core Problem
Most people's digital lives are fragmented across 3-5 ecosystems that don't communicate. iPhone with Apple Health, Google Calendar shared with family, Gmail, OneDrive from work, Garmin watch, banking apps. None of it talks to each other. Every app is an island. There is no single place to see and manage your full life.
The Solution
Life Connect sits on top of what users already have. It doesn't replace Google Calendar or Apple Health — it connects them all into one intelligent interface. The app reads and writes across every platform simultaneously. Zero migration. Zero data loss. The user's data stays in systems they already trust and pay for.
Key Differentiators
→Existing infrastructure as the backend. User's own Google Drive, iCloud, or OneDrive stores their data. No data liability. No per-user storage cost. Instant trust.
→Bidirectional sync. Not just a dashboard — reads AND writes back to source systems. Add to Google Calendar, check off tasks, log health data — all from one place.
→Intelligent onboarding. Q&A setup that detects what the user already uses and configures the app around their existing ecosystem automatically.
→Life-domain organization. Not organized by app — organized by life. Family. Health. Work. Finances. Trading. Kids. Personal growth. Each domain pulls from whatever tools feed it.
→AI layer built in. Review, coaching, analysis, and insights powered by AI — accessible within the same system where the data lives.
Target Market
→Families — shared calendar, kids development tracking, household management, spouse coordination
→Professionals — cross-platform work/life integration, performance tracking, goal management
→Health-focused users — fitness, nutrition, sleep, readiness scoring across Garmin, Apple Health, Google Fit
→Mixed-ecosystem users — the majority who use partial Apple + partial Google + partial Microsoft and have no unified view
Pricing Tiers
Trial — 30 days free, full access, no credit card required
Light — Core integrations, 3 domains, basic logging. ~$4.99/month
Pro — All integrations, unlimited domains, AI layer, family sharing, advanced analytics. ~$12.99/month

Concept & Vision
What it is, how it works, where it goes
Onboarding Flow
Step 1Q&A detects what ecosystem the user lives in — Apple, Google, Microsoft, or mixed
Step 2User selects their life domains — Family, Health, Work, Finance, Personal, Kids, etc.
Step 3App connects to their existing accounts — Google OAuth, Apple Sign-In, Microsoft login
Step 4System auto-configures based on detected tools and chosen domains
Step 5User is in their personalized Life Connect in under 10 minutes, with real data already populating
Integration Architecture
→Google — Calendar (read/write), Drive/Sheets (storage), Tasks (read/write), Gmail (read), Fit (read)
→Apple — Health (read via HealthKit), Calendar (read/write), iCloud Drive (storage), Sign-In
→Microsoft — Outlook Calendar (read/write), OneDrive (storage), Teams, To-Do
→Garmin / Fitbit / Whoop — health metrics, body battery, sleep, HRV (read)
→Financial — Plaid API bridges most major banks for read-only financial data
Why Existing Infrastructure is the Edge
User data lives in their own Google Drive, iCloud, or OneDrive. Life Connect never stores sensitive personal data on its own servers — it only stores connection tokens and preferences. This eliminates data liability, GDPR complexity per user, and per-user storage costs. It also eliminates the #1 objection to new productivity apps — "I don't want to move all my data." There is nothing to move. The user's existing infrastructure becomes the backend.
Long-Term Vision
Life Connect becomes the universal cockpit for personal and family life. As the user base grows, the AI layer gets smarter — pattern recognition across anonymized aggregate data to surface insights individual users can't see themselves. Enterprise version for small businesses. White-label version for health coaches, financial advisors, and life coaches to deploy to their clients. The platform that makes fragmented digital lives whole.

Legal & Protection
Patents, API terms, privacy, and risks
Idea Protection — Current Status
Conversations with Claude are private and not shared with other users. However, conversations are not a form of legal IP protection. The concept itself (unified life OS bridging ecosystems) is not patentable in the abstract — the specific implementation, onboarding methodology, and architecture are what can be protected. The best protection right now is execution speed.
Patent Strategy
When a working prototype exists: file a provisional patent application (~$320 USPTO fee for small entity). This grants 12 months of "patent pending" status. Potentially patentable: the Q&A onboarding system that auto-configures a personal OS based on detected user infrastructure, combined with the bidirectional sync architecture across competing ecosystems. Consult a patent attorney when prototype is ready.

GoogleMost open. APIs well-documented. Actively encourages ecosystem building. Low risk.

→Data processing agreements for EU users if marketing internationally

Key Risk: Existing Infrastructure as Backend
The biggest risk of the "user's own storage" model is API dependency. If Google changes its API terms, restricts third-party access, or charges for previously free endpoints, it directly impacts the product. Mitigation: build on the most stable, well-documented APIs. Diversify so no single platform is a single point of failure. Keep the app functional in read-only mode even if write access is restricted.
